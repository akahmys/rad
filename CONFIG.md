# rad 設定・配置ルールガイド

`rad` の動作設定、エクステンション、および関連アセット（スキル・ワークフロー等）の配置ルールと探索仕様について定義します。

## 1. 設定ファイル (`rad.json`)

`rad` の全体動作要件および各エクステンションの権限制限は、`rad.json` で一元管理されます。

### 1.1 設定ファイルの探索とマージ優先順位
`rad` コアの起動時、システムは以下の優先順位で設定ファイルを探索・ロードし、ローカル設定ファイルが存在する場合はそれをマージします。

1. **明示的指定**: `--config <PATH>` コマンドライン引数で指定されたパス。
2. **プロジェクトローカル（推奨）**: プロジェクトルート直下の `rad.json`。
3. **プロジェクトローカル（隠しフォルダ）**: プロジェクト内の `.rad/rad.json`。
4. **ユーザーグローバル**: 
   - Linux/macOS: `~/.config/rad/rad.json` (XDG Base Directory規格に準拠)
   - Windows: `%APPDATA%\rad\rad.json`

**ローカルオーバーライド（機密情報の分離）:**
上記のいずれかの設定ファイルがロードされた後、同じディレクトリ内に **`rad.local.json`**（例：プロジェクトルート直下の `rad.local.json` または `.rad/rad.local.json`）が存在する場合、コアはそれを自動ロードし、設定内容を**マージ（オーバーライド）**します。
> [!IMPORTANT]
> `rad.local.json` はAPIキーなどの個人機密情報を記述するためのローカル専用ファイルであり、リポジトリ共有を防ぐため必ず Git 管理外 (`.gitignore`) に設定してください。

### 1.2 設定ファイルのスキーマ例 (最低限の設定)
`rad.json` の記述は最低限でよく、省略されたパラメータは規定のデフォルト値が自動適用されます。以下はビルトインのオーケストレーターと外部の LLM モデルを設定する最小の `rad.json` 例です。

```json
{
  "version": "1.0",
  "extensions": [
    {
      "name": "standard-orchestrator",
      "source": "builtin://standard-orchestrator",
      "enabled": true,
      "config": {
        "model": "claude-3-5-sonnet-20241022"
      }
    }
  ]
}
```

### 1.3 機密情報の取り扱い
APIキーなどの機密情報を安全に扱うため、以下の方法でエクステンションへ引き渡します。

1. **環境変数の利用 (推奨)**:
   起動元のシェル環境変数（例：`export ANTHROPIC_API_KEY=sk-...`）を、エクステンションがシステムコール等を通じて直接取得します。設定ファイルにはキーを記述しません。
2. **ローカル設定ファイル (`rad.local.json`) の利用**:
   環境変数を汚したくないローカル開発環境では、以下のように `rad.local.json` に記述してマージさせます。

**`rad.local.json` の記述例:**
```json
{
  "extensions": [
    {
      "name": "standard-orchestrator",
      "config": {
        "api_key": "sk-ant-..."
      }
    }
  ]
}
```

### 1.4 省略可能パラメータとデフォルト値（規約）
`rad.json` で設定が省略された場合、コア内部で以下のデフォルトパラメータが自動適用されます。

| 設定項目 | デフォルト値 | 役割 |
| :--- | :--- | :--- |
| `core.workspace_dir` | `.` (実行ディレクトリ) | エージェントが作業を行う物理パスのルート。 |
| `core.snapshot_dir` | `.rad/snapshots` | ファイルシステムのスナップショットデータを退避・復元する場所。 |
| `core.log_dir` | `.rad/logs` | 動作ログおよびセッションログの出力先。 |
| `default_timeout.llm_stream_heartbeat_ms` | `15000` (15秒) | LLMのトークン受信パケット間隔の最大許容時間。 |
| `default_timeout.process_silent_timeout_ms` | `60000` (60秒) | `spawn_bash_process` で起動したコマンドが無通信（出力停止）になってからタイムアウトと見なす時間。 |
| `extensions[].permissions` | ビルトイン: フル権限 / 外部: 全制限 | 権限指定がない場合、外部エクステンションは全操作を拒否するセキュアデフォルトが適用される。 |
| `extensions[].config` | 空オブジェクト (`{}`) | エクステンション固有の設定。コアは中身を解釈せず、透過的（Passthrough）にエクステンションへ引き渡します。 |

## 2. ディレクトリ配置ルール

`rad` に関連するアセットやランタイムデータは、プロジェクト内の `.rad/` ディレクトリ（プロジェクトローカル）と、ユーザーのホームディレクトリ配下の `~/.rad/` ディレクトリ（ユーザーグローバル）に整理されます。

### 2.1 ディレクトリマップ

#### 2.1.1 プロジェクトローカル（プロジェクト個別の設定とデータ）
```text
<Project Root>/
├── rad.json                   # [推奨] プロジェクトローカル設定ファイル
└── .rad/                      # プロジェクト個別のデータ格納領域
    ├── rad.json               # [代替] プロジェクトローカル設定ファイル（隠し配置）
    ├── ext/                   # プロジェクト固有のローカルエクステンション (.wasm 等)
    │   └── analyzer.wasm
    ├── skills/                # プロジェクト固有のローカルカスタムスキル (スクリプト群)
    │   └── local_helper.sh
    ├── workflows/             # プロジェクト固有のローカルワークフロー
    │   └── local_flow.json
    ├── snapshots/             # コアが生成するスナップショット退避先
    │   └── <node_id>/
    └── logs/                  # 実行ログ出力先
```

#### 2.1.2 ユーザーグローバル（プロジェクトに従属しないアセット）
```text
~/.rad/                        # ユーザーグローバルのデータ格納領域
│                              # Windowsでは C:\Users\<User>\.rad\
├── ext/                       # 全プロジェクトで利用可能なグローバルエクステンション
│   └── global_analyzer.wasm
├── skills/                    # 全プロジェクト共通で使えるグローバルスキル
│   └── git_helper.sh
└── workflows/                 # 全プロジェクト共通のグローバルワークフロー
    └── standard_coding_flow.json
```

### 2.2 各ディレクトリの役割

#### 2.2.1 エクステンション (`.rad/ext/` および `~/.rad/ext/`)
`rad.json` の `extensions.source` に指定する Wasm ファイルやスクリプトを配置します。
- **プロジェクトローカル**: `./.rad/ext/` 配下
- **ユーザーグローバル**: `~/.rad/ext/` 配下
- **パッケージ構造 (ディレクトリ型)**:
  単一の `.wasm` だけでなく、設定やマニフェストを含むパッケージ形式での配置もサポートします。
  ```text
  .rad/ext/custom-analyzer/
  ├── extension.json           # エクステンションのメタデータ・デフォルト権限定義
  └── main.wasm                # 実行バイナリ
  ```

#### 2.2.2 スキル (`.rad/skills/` および `~/.rad/skills/`)
LLM（またはポリシー層）が特定の専門作業を行うためのツールやスクリプトを配置します。
- **プロジェクトローカル**: `./.rad/skills/` 配下
- **ユーザーグローバル**: `~/.rad/skills/` 配下
- コア起動時、エクステンションがこれら両方のディレクトリ内のスクリプト一覧とシステムプロンプト用のドキュメント（あれば）を収集し、LLMへツールとして提示します。
- LLMは、`spawn_bash_process` 経由で `.rad/skills/xxx.sh` などを呼び出します。

#### 2.2.3 ワークフロー (`.rad/workflows/` および `~/.rad/workflows/`)
開発プロセス定義や、DAG（Directed Acyclic Graph）の初期テンプレートを JSON/YAML で配置します。
- **プロジェクトローカル**: `./.rad/workflows/` 配下
- **ユーザーグローバル**: `~/.rad/workflows/` 配下

#### 2.2.4 スナップショット (`.rad/snapshots/`)
`take_snapshot` RPCによって退避されたファイルシステムの状態が、DAGの `node_id` ごとに格納されます（プロジェクトローカルのみ）。

#### 2.2.5 ログ (`.rad/logs/`)
コアおよびエクステンションの動作ログ、PTYセッションの入出力ログが保存されます。
※プロジェクト外で `rad` が実行された場合のグローバルログは `~/.rad/logs/` に保存されます。

## 3. 本体およびエクステンションの更新ルール

`rad` はシステム特権を必要とせず、ユーザー空間のみで更新・実行が可能です。

* **本体の更新**: `rad --update` コマンドを実行することで、公式リリースから最新の安定版バイナリがダウンロードされ、インプレースで置換されます。
* **エクステンションの更新**: `.wasm` やスクリプトファイルを差し替えるだけで動的に適用されます。
